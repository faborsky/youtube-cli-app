# YouTube CLI

Python CLI for managing YouTube videos via YouTube Data API v3.

## Quick Start

```bash
source venv/bin/activate && python youtube_cli.py auth    # First time only
source venv/bin/activate && python youtube_cli.py <command>
```

## Structure

```
youtube_cli.py               # Main CLI (single file)
.env                         # YOUTUBE_CLIENT_ID, YOUTUBE_CLIENT_SECRET
~/.youtube-cli-token.json    # OAuth credentials (auto-generated by auth command)
```

## Commands

| Command | Purpose | Key Flags |
|---------|---------|-----------|
| `auth` | One-time OAuth setup (opens browser) | -- |
| `videos` | List recent videos | `--limit`, `--json` |
| `video-get` | Get video details | `--video-id ID`, `--json` |
| `upload` | Upload video + metadata | `--file`, `--title`, `--description`, `--tags`, `--category`, `--privacy`, `--thumbnail`, `--playlist`, `--language`, `--publish-at`, `--json` |
| `video-update` | Update video metadata | `--video-id ID`, `--title`, `--description`, `--tags`, `--privacy`, `--thumbnail`, `--json` |
| `video-delete` | Delete video | `--video-id ID`, `--confirm` (required) |
| `stats` | Channel/video statistics | `--video-id` (optional), `--json` |
| `playlists` | List playlists | `--json` |

## Safety

- Videos are uploaded as **private** by default.
- Delete requires `--confirm` flag.
- Credentials are in `.env` (never commit this file).
- OAuth token stored in home directory (`~/.youtube-cli-token.json`), not in project.

## API Notes

- Uses YouTube Data API v3 via `google-api-python-client`.
- Auth: OAuth 2.0 Installed App flow. One-time browser consent, then auto-refresh via stored refresh token.
- Upload: Resumable upload with chunked transfer (~12.5 MB chunks) and exponential backoff retry.
- `video-update` fetches current data first, then merges changes (YouTube API requires full snippet on update).
- Thumbnail upload is a separate API call after video upload.
- Default quota: 10,000 units/day. Upload costs ~1,600 units, read = 1 unit, write = 50 units.

## Conventions

- All commands support `--json` for machine-readable output.
- The CLI exits with code 1 on errors and prints errors to stderr.
- Progress messages go to stderr, results to stdout.
